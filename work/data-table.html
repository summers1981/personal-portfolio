Back to work Component Design Work in progress

Improving data clarity and
interaction at scale

The data table is one of the most used components in the product. But it had gaps — missing sticky headers, unclear sorting affordances, confusing expandable rows, and weak support for density-heavy workflows. Fixing it meant building a component that could serve reporting, bulk actions, and index views consistently.

Scope Component Redesign
Context FreeAgent · Cross-product
Category Component Design · Data
Focus Usability · Density · Interaction
Objective

A table component for sophisticated workflows

The goal was to deliver a redesigned data table component that supports the company's premium product positioning — enhancing data comprehension, improving interaction affordances, and reducing friction in report-heavy and bulk-action workflows.

The component needed to work across reports and index views while establishing consistency across all product teams.

Challenges & Goals

A component with growing gaps

  • Missing critical features: sticky headers, density adjustment, consistent sorting
  • Unclear interaction affordances for expandable rows and horizontal scrolling
  • Multiple links per row creating user confusion about primary actions
  • Weak visual hierarchy for totals and grouped data
  • Product teams lacked capacity to prioritise this work independently
  • Needed to be flexible enough for both reports and index views
Solutions

Built for density, clarity, and control

  • Sticky headers for context retention in lengthy tables
  • Density controls supporting standard and compact viewing modes
  • Enhanced sorting indicators with clearer visual feedback
  • Monospace numerals improving alignment and readability
  • Improved expandable row affordance and comprehension
  • Bold total dividers for easier identification of summary rows
  • Scrolling cues via edge indicators on narrow viewports
  • Refined row selection supporting individual and bulk actions
  • Link improvements: underlines on hover only, single primary link per row
  • Responsive and fully accessible with keyboard and screen reader support
Process

Research, design, test, implement

1

User Research & Audit

Leveraged prior testing and conducted a product-wide table usage audit to identify the highest-impact improvements.

2

Design & Prototyping

Led cross-team redesign with prototypes validating affordance and interaction changes before any engineering investment.

3

Testing & Validation

Confirmed hypotheses about link confusion, density preference, and sorting and header improvements through structured usability testing.

4

Implementation & Documentation

Team built the component with comprehensive FreeStyle documentation covering usage, accessibility, and interaction guidelines.

Results & Impact

Clarity, control, and confidence

📌

Sticky Headers

Users retain column context while scrolling through large datasets — a significant improvement for accountants and power users.

🔤

Better Numerical Legibility

Monospace numerals and refined spacing make it easier to align and compare figures across rows.

🎛️

Density Controls

Standard and compact viewing modes give users control over how much data they can process at once.

✅

Validated by Testing

User testing confirmed improvements to link clarity, sorting affordance, and expandable row comprehension.

Outcome

A redesigned table component serving premium users with better data comprehension, clearer interactions, and consistent implementation across the product.
